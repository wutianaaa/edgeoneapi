## 🚀 高级功能

### 1. 流式输出停止功能
- ✅ 在 AI 生成回复时可随时停止
- ✅ 使用 AbortController 中止请求
- ✅ 发送中显示停止按钮（方块图标）

**使用方法：**
当 AI 正在生成回复时，点击右下角的停止按钮（方块图标）即可中止生成。

### 2. 模型参数配置
- ✅ Temperature 参数调节（0-2，滑块控制）
- ✅ Max Tokens 配置（1-32000）
- ✅ Top P 参数调节（0-1，滑块控制）
- ✅ 参数自动保存到 localStorage
- ✅ 实时显示当前参数值

**使用方法：**
1. 点击右上角的滑块图标打开设置面板
2. 调节各项参数
3. 参数会自动保存并应用到下一次请求

**参数说明：**
- **Temperature**: 控制输出的随机性（0=确定性，2=高随机性）
- **Max Tokens**: 限制生成的最大 token 数量
- **Top P**: 核采样参数，控制输出的多样性

### 3. 聊天导出功能
- ✅ 导出为 Markdown 格式
- ✅ 导出为 JSON 格式
- ✅ 包含会话标题、时间戳
- ✅ 包含思考过程（如果有）
- ✅ 包含性能统计数据

**使用方法：**
1. 点击右上角的滑块图标打开设置面板
2. 在"导出对话"部分选择格式：
   - **Markdown**: 适合阅读和分享，支持折叠的思考过程
   - **JSON**: 完整的结构化数据，可用于备份或程序处理

**导出文件命名：**
- Markdown: `chat-{session_id}-{timestamp}.md`
- JSON: `chat-{session_id}-{timestamp}.json`

### 4. 批量操作功能

#### 渠道批量导入/导出
- ✅ 导出所有渠道配置（不含 API Key）
- ✅ 导出所有模型映射
- ✅ 批量导入渠道和模型
- ✅ 导入时自动生成新 ID

**使用方法：**
1. 在"渠道管理"页面点击"导出"按钮
2. 保存 JSON 文件
3. 点击"导入"按钮选择 JSON 文件
4. 系统会自动创建所有渠道和模型

**导出格式示例：**
```json
{
  "version": "1.0",
  "exported_at": "2026-06-03T10:30:00.000Z",
  "channels": [
    {
      "name": "openai-primary",
      "type": "openai",
      "base_url": "https://api.openai.com/v1",
      "enabled": true,
      "weight": 1
    }
  ],
  "models": [
    {
      "model": "gpt-4o-mini",
      "upstream_model": "gpt-4o-mini",
      "channel_ids": ["ch_xxxxx"]
    }
  ]
}
```

#### 用户批量导入/导出
- ✅ 导出所有用户配置（不含 API Key）
- ✅ 批量导入用户
- ✅ 导入时自动生成新 API Key

**使用方法：**
1. 在"用户密钥"页面点击"导出"按钮
2. 保存 JSON 文件
3. 点击"导入"按钮选择 JSON 文件
4. 系统会自动创建用户并生成新的 API Key

**注意事项：**
- 导出不包含 API Key（安全考虑）
- 导入时会自动生成新的 API Key
- 导入后需要记录新生成的 API Key

**导出格式示例：**
```json
{
  "version": "1.0",
  "exported_at": "2026-06-03T10:30:00.000Z",
  "users": [
    {
      "name": "default-user",
      "allowed_models": ["gpt-4o-mini"],
      "is_default": false,
      "enabled": true
    }
  ]
}
```

## 📊 技术实现细节

### 深色模式
- 使用 CSS 变量实现主题切换
- `:root.dark` 类控制深色模式
- 所有颜色通过 CSS 变量定义，确保一致性

### 停止功能
- 使用 `AbortController` API
- 在请求时传递 `signal` 参数
- 支持流式和非流式请求的中止

### 参数配置
- 参数存储在 `localStorage`
- 在发送请求时包含在 payload 中
- 支持实时预览参数值

### 批量操作
- 使用原生 File API 读取文件
- JSON 格式保证数据完整性
- 导入时进行格式验证
- 错误处理：单个项目失败不影响其他项目
